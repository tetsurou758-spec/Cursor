### ② フォローコール（ヘッダー変更: `FC-STS` → `ﾌｫﾛｰｺｰﾙ`）
- `followcall_status` カラムを `NULL（未実施）/ 日付文字列（実施済）` 形式に変更

## バグ修正

### Fix 1: 満期一覧で検索結果が表示されない
- **原因**: `memo` 変数を `memoVal` にリネームした際、detail展開行の `${memo||...}` の更新漏れ
- **症状**: `ReferenceError: memo is not defined` で `renderTable` 全体がクラッシュ
- **修正**: `${esc(memoVal)||'（なし）'}` に修正

### Fix 2: フォローコール更新エラー（旧サーバー起動問題）
- **原因**: サーバーが古いコードで起動したままで PUT routes 未登録 → 404
- **修正**: 全Pythonプロセスを終了して最新コードでサーバーを再起動

### Fix 3: 更改STS「満期落ち」選択肢の誤り
- **原因**: 更改STS上の表示は「落ち」が正しく、「満期落ち」はコンタクト履歴の種別
- **修正**: プルダウンから「満期落ち」を削除、「落ち」選択時にコンタクト履歴へ種別「満期落ち」で登録

### Fix 4: ポップオーバーの保存が全て失敗する（最重要）
- **原因**:
  ```
  openMemoPopover(event, 1450, '')
    → _popContractId = 1450    ← セット
    → openPopover('pop-memo', event)
        → closePopover()        ← _popContractId = null にリセット ← バグ
  saveMemo() 実行時: _popContractId = null
  → PUT /api/maturity/null/memo → 422 Unprocessable Entity
  ```
- **発見方法**: Playwrightでブラウザ動作を自動再現し、コンソールログで `contractId= null` と 422 エラーを特定
- **修正**: 3関数全て（openMemoPopover / openFcPopover / openRenewalPopover）で
  `_popContractId` / `_popCustomerId` の代入を `openPopover()` 呼び出しの**後**に移動
- **Playwright検証結果**:
  - memo: contractId=1450 → 200 OK・DB反映確認
  - followcall: contractId=1450, date=2026-06-05 → 200 OK・DB反映確認
  - renewal: contractId=1451, status=対応中 → 200 OK・DB反映確認
