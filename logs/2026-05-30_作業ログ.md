**新規追加テーブル：**

| accidents | 2件（M-A001-001・M-A001-003） |

| ロール | 利用可能機能 |
|---|---|

### バックエンド（backend/main.py 改修）

**ログインAPI改修：**
- JWT に `role_id` / `agency_id` / `group_code` を追加
- レスポンスに `role_name` / `permissions`（feature_codeリスト）を追加

**新規APIエンドポイント：**

| メソッド | パス | 認証 | 内容 |
|---|---|---|---|
| GET | `/api/permissions` | JWT | role_idの利用可能feature_codeリスト・role_nameを返す |
| GET | `/api/users` | JWT（管理者のみ） | 自代理店ユーザー一覧をロール情報とともに返す |
| POST | `/api/users` | JWT（管理者のみ） | 新規ユーザー登録 |
| PUT | `/api/users/{user_id}` | JWT（管理者のみ） | ユーザー情報・ロール変更 |
| DELETE | `/api/users/{user_id}` | JWT（管理者のみ） | 論理削除（is_active=0） |
| GET | `/api/role-matrix` | JWT（管理者のみ） | ロール×機能の権限マトリクスを返す |

**GET /api/maturity 改修：**
- accidents テーブルを EXISTS サブクエリで JOIN し `has_accident` を動的に算出
- `group_code` で同グループ代理店の契約もまとめて参照可能に

## デバッグ・修正記録

### 修正1：session.js の const API_BASE 重複宣言（SyntaxError）

**発生場所：** `frontend/js/session.js` L8  
**症状：** ブラウザでダッシュボード・満期管理画面のデータが一切表示されない  
**原因：**  
session.js で `const API_BASE = 'http://127.0.0.1:8000'` を宣言していたが、この変数はsession.js内で一切使用されていなかった。一方、dashboard.html / staff_dashboard.html / maturity.html / staff_maturity.html の4ファイルのインラインスクリプトも同名の `const API_BASE` を宣言しており、ブラウザのglobal lexical environmentで重複宣言となり `SyntaxError: Identifier 'API_BASE' has already been declared` が発生。インラインスクリプト全体が実行されなくなっていた。

**修正内容：** session.js から未使用の `const API_BASE` 宣言を削除（2行削除）

**確認済み動作（curl検証）：**

| テスト | 結果 |
|---|---|
| POST /api/login（A001/admin） | 200 OK ✓ |
| GET /api/dashboard（代理店A001） | 今月247件・来月325件 ✓ |
| POST /api/staff/login（S001） | 200 OK ✓ |
| GET /api/dashboard（社員S001・全代理店7社） | 今月601件・来月856件 ✓ |
| GET /api/dashboard（社員S002・X001部課5社） | 今月601件・来月856件 ✓（B001/C001はデータなし） |
| GET /api/dashboard（社員S003・Y001部課1社） | 今月0件・来月0件 ✓（B001はデータなし） |
| GET /api/maturity（buka_code返却確認） | buka_code=X001 正常返却 ✓ |
